# 開発の工程
*  要件定義前(企画)
   * アイデアを書き出す(実現可能性は考慮しない) 
   * 企画書(できればデジタルデータ)
* 要件定義
  * 要件定義書
    * 解決・達成したい課題に必要であろう機能を洗い出す 
* 基本設計
  * 機能一覧の作成
    * 要件定義で上げた機能をより詳細に記す
  * 画面設計
    * 必要な画面とそのつながりを記す
    * 各画面ごとの表示項目や機能を考える
  * データ設計
    * テーブルの攻勢を考える
    * API設計
      * 各エンドポイントで提供するデータや機能を考える
* モック画面の作成(HTML, CSS)
* 詳細設計
  * Spring Bootの設計
  * クラス設計
  * コンポーネント設計
  * データベース設計
* 実装(プログラミング)
* テスト
* 運用・保守

## 要件定義前
* 提案書
  * 解決すべき課題
  * 課題解決のための仕様 
  * 納期

## 要件定義
* システム化の範囲
  * 現在の業務内容や業務の流れ(どのような課題があるか)
  * システム導入後の業務の流れ(どのように解決するか)
* 機能要件(課題解決のためにシステムに必要な機能)
  * 
* 非機能要件(性能、セキュリティ、バックアップ等)
  * 

## 設計(定義された要件をシステムとしてどのように実現するかを検討)
* 画面設計
  * どのような画面が必要か
  * それらがどうつながるか
* 機能設計
  * 必要な機能を抽出
* データ設計
  * システムに必要なテーブルやカラムを検討
* コード設計
  * 具体的なクラスやメソッドを検討
